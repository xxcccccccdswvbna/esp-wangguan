在 Home Assistant (HA) 中实现“自动化任务直接发送指定的抓包命令（HEX 数据）”是一个非常高级且实用的玩法！比如你可以设置：“当室内温度高于 28℃ 时，自动发送开风扇的抓包命令”。

要实现这个功能，有两种最优雅的方式。**强烈推荐方式一**，因为它不需要 MQTT，所有固件（VT1~VT4）通用，且与 HA 原生集成度最高。

---

### 🌟 方式一：使用 ESPHome 原生自定义服务（最推荐，所有固件通用）

我们可以在 Python 脚本生成的 YAML 中，向 HA 注册一个**自定义服务**。这样 HA 就会多出一个类似 `esphome.vt1_send_raw_hex` 的动作，你在自动化里直接调用它并传入抓包数据即可。

#### 第一步：修改 Python 脚本，注册自定义服务

打开你的 `generate_device_table.py`，找到 `base_template` 变量（大概在文件中间部分），找到 `api:` 这一节。

**修改前：**
```python
api:
  reboot_timeout: 0s
```

**修改后（增加 `services` 定义）：**
```python
api:
  reboot_timeout: 0s
  # 🔥 核心新增：向 HA 注册一个发送原始 HEX 抓包命令的自定义服务
  services:
    - service: send_raw_hex
      variables:
        hex_data: string
      then:
        - lambda: |-
            id(ct1_ble).send_hex(hex_data);
```

*(注：修改后 `git push` 重新编译刷入固件，HA 重启后就会识别到新服务。)*

#### 第二步：在 Home Assistant 中编写自动化

刷入新固件后，HA 会自动生成一个服务，名字通常是 `esphome.<你的设备名>_send_raw_hex`（例如 `esphome.vt1_send_raw_hex`）。

**自动化 YAML 样板：**
```yaml
alias: 温度过高自动发送抓包命令开风扇
description: 当客厅温度 > 28度时，直接发送风扇开启的抓包HEX数据
triggers:
  - trigger: numeric_state
    entity_id: sensor.temperature
    above: 28
conditions: []
actions:
  # 🔥 调用 ESPHome 自定义服务
  - action: esphome.vt1_send_raw_hex  # <--- 请根据你实际的设备名修改，如 esphome.ct1_send_raw_hex
    data:
      # 直接粘贴你抓包的 HEX 数据（支持单包，也支持用 | 分隔的多包）
      hex_data: "0201021BFF114D19184880277000000181972E277070005FC86FB84848A7FB|0201021BFF114D191F488027700000015EC86FB84848A7F080809727277072"
mode: single
```

**💡 优势**：
1. **无需 MQTT**：走 ESPHome 原生 API 通道，最稳定，延迟极低。
2. **支持多包**：`hex_data` 里你可以直接粘贴用 `|` 隔开的多个抓包数据，网关的智能队列会自动帮你排队发送。
3. **HA 界面友好**：在 HA 自动化界面选择“执行动作”时，下拉菜单里直接就能搜到 `ESPHome: vt1 send raw hex`，输入框里直接粘 HEX 即可。

### 📡 方式二：通过 MQTT 发送（仅限 VT2 / CT2 固件）

如果你刷的是带 MQTT 功能的固件（VT2），你的 ESP32 已经订阅了 `vt2/ble/send` 这个主题。你可以直接通过 MQTT 发送抓包命令。

**自动化 YAML 样板：**
```yaml
alias: 定时发送抓包命令关灯
description: 每天晚上 11 点，通过 MQTT 发送关灯的抓包命令
triggers:
  - trigger: time
    at: "23:00:00"
conditions: []
actions:
  # 🔥 调用 HA 原生的 MQTT 发布服务
  - action: mqtt.publish
    data:
      topic: "vt2/ble/send"  # <--- 对应 VT2 固件的 MQTT 接收主题
      payload: "0201021BFF114D191A4880277000000126277670005FC86FB84848A7F08088|0201021BFF114D19174880277000000181809727277070005FC86FB84848A5"
      retain: false
mode: single
```

**💡 优势**：
不需要修改任何 Python 脚本和 ESPHome 代码，只要刷了 VT2 固件就能直接用。

### 🧠 进阶玩法：结合 HA 变量动态发送

如果你有很多抓包命令，不想每次都硬编码在自动化里，可以在 HA 里创建一个 **Input Text (输入框文本)** 或 **Input Select (下拉菜单)**，把抓包命令存进去，自动化直接读取。

**示例：使用下拉菜单选择抓包命令**
1. 在 HA 的 `configuration.yaml` 中定义一个下拉菜单：
   ```yaml
   input_select:
     ble_raw_commands:
       name: "BLE 抓包指令库"
       options:
         - "0201021BFF...开灯包1"
         - "0201021BFF...关灯包1"
         - "0201021BFF...风速1包1"
   ```
2. 自动化直接读取这个下拉菜单的值发送：
   ```yaml
   actions:
     - action: esphome.vt1_send_raw_hex
       data:
         hex_data: "{{ states('input_select.ble_raw_commands') }}"
   ```
   这样你就建立了一个**专属的抓包命令武器库**，自动化想发什么就发什么！

### 🚀 总结建议

1. 如果你希望**所有网关（VT1~VT4）都能接收自动化指令**，请采用 **方式一**，修改 Python 脚本加入 `services`。
2. 抓包数据在填入 HA 时，**不需要带 `0x` 前缀**，直接填纯 HEX 字符串（如 `0201021BFF...`）。
3. 如果需要发送双包/三包，直接用 `|` 连接（如 `包1|包2`），你的网关底层已经完美支持这种解析。

去修改 Python 脚本并刷入固件吧，解锁这个功能后，你的 BLE 网关就真正变成了一个“万能红外/射频发射器”！
